# Guia de Contribuição — PortfolioHUB

Obrigado por contribuir com o PortfolioHUB! Siga as orientações abaixo para manter o projeto organizado.

## Fluxo de Trabalho

1. **Nunca faça push direto na branch `main`**
2. Crie sempre uma branch nova para sua alteração:
   ```bash
   git checkout -b feature/nome-da-sua-feature
   ```
3. Faça seus commits seguindo o padrão abaixo
4. Abra um **Pull Request** para revisão antes do merge

## Padrão de Commits (Conventional Commits)

| Prefixo | Quando usar |
|--------|-------------|
| `feat:` | Nova funcionalidade |
| `fix:` | Correção de bug |
| `docs:` | Alteração em documentação |
| `style:` | Mudança visual (CSS, layout) |
| `refactor:` | Refatoração de código |

**Exemplo:**
```
feat: adiciona seção de projetos na página inicial
```

## Abrindo um Pull Request

- Descreva o que foi alterado e por quê
- Aguarde ao menos **1 aprovação** antes de fazer merge
- Não aprove seu próprio PR

## Dúvidas

Abra uma **Issue** no repositório descrevendo o problema ou sugestão.
